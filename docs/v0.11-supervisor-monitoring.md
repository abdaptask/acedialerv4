# v0.11.x — Supervisor Call Monitoring

This document captures the scope, design decisions, and sequencing for
adding Pulse-equivalent supervisor call monitoring to ACE Dialer.

Source feature spec: Telnyx Call Monitoring API PDF (uploaded
2026-06-15). Five capabilities documented: Listen, Whisper, Barge,
Take Over, Real-Time Queue Analytics.

## Decisions locked (2026-06-15)

| Decision | Choice | Rationale |
|---|---|---|
| Supervisor role model | New `isSupervisor` flag on User, separate from `isAdmin` | Lets admins manage settings without auto-listening to calls; lets team leads supervise without admin |
| Agent disclosure when monitored | None — silent monitoring | Matches Telnyx default; defensible under typical employer monitoring policies. HR review recommended before rollout |
| Take Over scope | Barge + mute agent only | Avoids customer-leg transfer complexity in v0.11.1; covers the "I'm taking over this conversation" use case |
| Session recording | None — audit log only | No audio storage; database row records who/whom/when/duration. Lowest legal and storage cost |

## Release sequence

### v0.11.0 — Foundation (already planned)

**Scope:** Presence + DND + Voicemail Retention.

Why this is a precondition: the Supervisor view in v0.11.1 needs a
live list of who is on a call right now. That list is built by the
Presence work in v0.11.0. Pulling Presence forward avoids
re-implementing the same wheel.

### v0.11.1 — Supervisor monitoring suite (bundled)

**Scope:** Listen + Whisper + Barge + Take Over in a single release.
All four use the same Telnyx Dial endpoint with different
`supervisor_role` parameter values, so bundling adds minimal overhead.

**Schema additions:**
```prisma
model User {
  // ... existing fields ...
  isSupervisor Boolean @default(false) @map("is_supervisor")
}

model MonitorSession {
  id                       Int      @id @default(autoincrement())
  supervisorId             Int
  supervisor               User     @relation("MonitorSupervisor", fields: [supervisorId], references: [id])
  agentUserId              Int
  agent                    User     @relation("MonitorAgent", fields: [agentUserId], references: [id])
  agentCallControlId       String
  customerCallControlId    String?
  role                     MonitorRole
  startedAt                DateTime @default(now())
  endedAt                  DateTime?
  durationSec              Int?

  @@index([supervisorId, startedAt])
  @@index([agentUserId, startedAt])
}

enum MonitorRole {
  LISTEN
  WHISPER
  BARGE
  TAKEOVER
}
```

**API surface:**
```
GET  /supervisor/active-calls
       -> [{ agentUserId, agentName, customerNumber, lineLabel,
             callControlId, startedAt, durationSec }, ...]
POST /supervisor/monitor
       body: { agentCallControlId, role: 'LISTEN' | 'WHISPER' | 'BARGE' }
       -> { sessionId }
       Composes the Telnyx dial with the supervisor_role param.
POST /supervisor/take-over
       body: { agentCallControlId }
       -> { sessionId }
       Composable: dial as barge -> POST /v2/calls/{agentCallId}/actions/mute
POST /supervisor/end-session/:sessionId
       -> hangs up supervisor leg + finalizes audit log
```

**Authorization:** All `/supervisor/*` routes gated by
`request.user.isSupervisor === true`. Non-supervisors get 403.

**Web UI:**
- New "Supervisor" item in main nav, visible only when `isSupervisor`
- Active-calls list: agent name, customer number, call duration, line,
  four action buttons per row (Listen / Whisper / Barge / Take Over)
- Real-time updates via existing socket service (subscribe to
  `presence:call-started` and `presence:call-ended` from v0.11.0)
- Supervisor's own in-progress monitor session shows at the top with
  an "End monitoring" button

**Audit log:**
- Settings → Audit Log gets a "Monitor sessions" tab (admin-only)
- Filterable by supervisor, agent, date range
- Columns: supervisor, agent, customer number, role, started, duration

### v0.11.2 (optional) — Queue analytics

Only ship if a decision is made to migrate from per-user routing to
Telnyx call queues. Adds queue depth + average wait time displays in
the supervisor view. Agent availability is already covered by v0.11.0
Presence. Abandoned call counts come from `max_wait_time_secs` expiry
events on our webhook handler.

## Audio routing — non-issue

Worth noting because it could have been an issue: WebRTC audio mixing
is handled server-side by Telnyx. When the supervisor dials in with a
`supervisor_role`, Telnyx mixes the appropriate streams (agent only
for Whisper, agent+customer for Listen/Barge) and the supervisor's
dialer receives a single pre-mixed Opus stream just like any other
call. No special browser-side WebRTC plumbing needed.

## Open questions (need answers before v0.11.1 implementation starts)

1. **Telnyx tier check** — confirm `supervisor_role` parameter is
   available on our current Telnyx account plan. The PDF documents
   it as "Native" but doesn't list tier requirements.
2. **HR / legal review** — silent monitoring requires either employee
   acknowledgement (handbook clause) or explicit consent. Confirm
   ApTask's handbook covers this before rolling v0.11.1 to users.
3. **Who gets `isSupervisor=true` initially?** — needs a list from
   admin. Probably abdulla + team leads. Mass-toggle via SQL or
   one-off admin UI? (Likely just SQL for the initial set.)

## Out of scope for v0.11.x

- Multi-tenant supervisor scoping (one supervisor sees only one team's
  agents) — defer to v0.12.x if needed
- Mobile supervisor view (mobile dialer is read-only today; supervisor
  features stay desktop-only for v0.11.1)
- Outbound call monitoring (Pulse only supervised inbound; we match
  that for v0.11.1)
- Pre-call queueing UI for agents ("I'm ready, queue me up")
